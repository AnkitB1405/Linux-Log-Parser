Structure explanation:
parser has submodules to help the main.py program like syslog parser, network parser, ssh parser, etc
tests are there to check with a fixed set of logs and check if the parser is working correctly
sample_logs are to see how the program behaves when you change it
outputs are a place to store important results for future watch ( will mostly be in gitignore)

Development Stage:
convert logs to cleaned json formats with only necessary data.
obtain real time sample logs from wherever available
build parser modules
build test modules
keep storing important results in outputs
impliment a detection system by flagging suspicious activity, based on (idk fill in the blanks)
maybe pass it on onto an ai agent to analyze log pattern, attack behaviour, source, etc. 

two versions of the project
v1: just for the homelab. Monitors ssh, network, syslog, docker logs. Detects using algorithm and flags alert. 
v2: industry grade. Monitors ssh, network, syslog, apache, etc. Parser logs, detects suspiciour activity and forwards to an AI model to analyze and report the activity.
